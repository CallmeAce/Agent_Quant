## MVP 功能范围优先级（与 BRD 对齐）

本文件对应 todo「根据业务目标和资源确定MVP必须实现的前端/后端/Agent功能清单」。

---

## 1. 核心业务闭环（MVP 必须打通）

1. **数据 → 策略 → 信号 → 用户展示与通知** 四步闭环：
   - 从 QMT + Tushare 拉取数据（行情、财报、账户）。
   - 执行至少 1–2 个示例策略的 **选股与调仓逻辑**。
   - 生成 T0/T+1 的买卖点信号。
   - 用户在前端 Dashboard 可查看净值/持仓/信号，并通过 Telegram/邮件接收通知。

MVP 所有功能均围绕这个闭环服务。

---

## 2. 前端（用户端）MVP

### 2.1 必须实现

- 登录/注册/退出（手机号或邮箱 + 密码）。
- 策略列表（策略广场）：
  - 展示策略名称、风格标签、近 1Y/全回测收益、最大回撤、订阅人数（可选）。
- 策略详情页（精简版）：
  - 净值曲线（至少展示“全周期 + 近 3 个月”两个区间）。
  - 风险指标摘要（年化收益、最大回撤、夏普）。
  - 最近 N 笔信号列表（时间、方向、标的数）。
- 用户 Dashboard：
  - “我的策略”列表（已订阅）。
  - 单策略页：当前标的列表（来自策略层模拟持仓）、最近信号列表。

### 2.2 可延后

- 策略组合视图（跨策略聚合净值和风险）。
- 高级回测细节展示（因子暴露、风格归因等）。
- 发票/报销信息管理。

---

## 3. 前端（管理员端）MVP

### 3.1 必须实现

- 管理员登录。
- 策略管理：
  - 策略列表与状态（草稿/内测/公开/下线）。
  - 基础信息编辑（名称、标签、市场、调仓频率等）。
- 回测入口：
  - 能够针对指定策略/参数/时间区间触发回测任务。
  - 查看简要回测结果（核心绩效指标 + 净值曲线缩略图）。
- 订阅与用户概览：
  - 用户数量、按策略聚合的订阅数/收入（简单统计即可）。

### 3.2 可延后

- 细粒度因子管理与分析 dashboard。
- 策略版本对比与 A/B 测试配置。
- 复杂运营配置（优惠券、阶梯定价等）。

---

## 4. 后端 API 服务 MVP

### 4.1 必须实现

- `AuthService`：
  - 注册、登录、Token 校验。
- `StrategyService`：
  - 策略元数据查询（列表/详情）。
  - 策略绩效数据查询（净值曲线、指标）。
  - 策略信号查询（按策略/时间）。
- `BacktestService`：
  - 创建回测任务。
  - 查询回测状态与基础结果。
- `SignalService`：
  - 生成每日/每周策略信号的内部接口（供 Agent 调用）。
  - 查询历史信号。
- `NotificationService`：
  - 配置用户的 Telegram/邮箱。
  - 发送信号通知 & 记录发送结果。
- 支付回调（最小化实现）：
  - 接收微信/支付宝回调，更新 `order_payment` 与 `subscription` 状态。

### 4.2 可延后

- 因子分析 API（暴露 IC、分层收益等）。
- 高级归因分析 API。
- 完整运营报表导出接口。

---

## 5. ETL 与数据平台 MVP

### 5.1 必须实现

- Tushare：
  - `ods_daily_bar`（日线行情）。
  - `ods_financial_statement`（核心财报字段）。
  - `dim_security` / `dim_trade_calendar` / 行业映射。
- QMT：
  - `ods_account_equity_daily`（账户净值）。
  - `ods_position_snapshot`（日终持仓）。
  - `ods_trade`（成交）。
- 因子：
  - 一小批关键因子（估值/成长/动量/波动/流动性），并写入 `dwd_factor_value`。

### 5.2 可延后

- 分钟级行情。
- 更细的资金流水与风险暴露表。

---

## 6. Openclaw Agent（QuantOps）MVP

### 6.1 必须实现

- 定时任务：
  - 每个交易日 T0 收盘后：触发 Tushare 日线拉取 + 因子计算 + 策略选股任务。
  - 每个交易日 T+1 开盘前/后：触发实盘信号生成任务。
- 健康检查：
  - 校验核心 ODS 表当天是否有数据（`ods_daily_bar`、`ods_trade` 等）。
  - 校验任务执行结果中记录数是否在合理区间（如选股数量>0）。
- 告警：
  - 当任务失败/数据缺失/结果异常时，向管理员 Telegram/邮件发送告警。

### 6.2 可延后

- 自动生成日报/周报/月报（初期可通过手工脚本完成）。
- 因子体检与策略下线建议自动化。

---

## 7. 实施优先顺序（粗粒度）

1. **数据与因子层**：表结构、ETL 骨架、核心因子计算。
2. **策略引擎与信号**：选股/调仓逻辑、信号表落地。
3. **后端 API**：为前端提供查询接口，为 Agent 提供内部接口。
4. **前端 MVP 页面**：用户端 > 管理端。
5. **Agent 调度与告警**：打通生产任务流。
